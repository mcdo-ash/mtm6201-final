Reflection: I found this to be a very interesting final assignment. Rather than building from a mockup we were provided, this was a mockup we had created in another course. This made me not only reflect on how I write my code, but also how I design my mockup designs and how I can make them more in line with how it would be done with code.

With that in mind, there were some adjustments that were made. Some on purpose, and some were due to a misunderstanding of how to accomplish some of the things that were in figma. Some of the instantly noticable changes were: The more success buttons taking up the bottom of the screen, the tablet layout being different for events and donation, the desktop events and donation page, the monetary donations button color, and the more products button colors. 

For the layout changes, some of the ways that I had laid it out on figma didn't make as much sense when I actually began to code it. Some elements were not aligned but not for any stylistic reason, they just weren't. For other layouts such as the tablet events page, I had opted to make it flow top to bottom since that made more sense for a (although larger) mobile layout. 

For the more success buttons, I had played around with making it actually sit in the middle, but I had come to like how it took up the remaining space on the bottom of the screen so I left it. 

For the changing of the buttons colors, after putting the website through the WAVE accesibility checker, it had come up as poorly contrasted. So I changed them to be darker so it would adhear to a more accessible design.

Overall, I recognize there are some decent design flaws and its not the most impressive looking website. After stressing over all the small details for hours, and becoming crippled by how I felt like I wouldn't be able to create anything I would be satisfied with, I remembered some wise words: "Don't let the good get in the way of the perfect.". From that statement I have been able to create something I feel is something I can be happy with.
